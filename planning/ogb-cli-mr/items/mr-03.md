## What to build

Implement **`ogb mr status <number>`** combining merge request lifecycle state with **mergeability** from the mergeability endpoint. Human output: one-line state plus mergeability summary. JSON output: `status`, `isDraft`, approval counts, mergeability `status` and `message`.

## Acceptance criteria

- [ ] Fetches MR detail and mergeability in one command invocation
- [ ] Human output suitable for script grep (state name + mergeability hint)
- [ ] `--json` includes all fields agents need to gate on merge readiness
- [ ] Does not include pipeline CI status (out of scope)
- [ ] Unit tests with mocked HTTP for Approved + mergeable vs blocked cases
