## What to build

Implement **`ogb mr diff <number>`** to fetch unified diff data from the changes endpoint. Human output: readable patch-style or per-file summary (match issue command verbosity). **`--json`** emits the structured changes payload (files, hunks, lines) for agent consumption.

## Acceptance criteria

- [ ] Calls GET `…/merge-requests/{n}/changes`
- [ ] Human diff is usable in terminal for quick review
- [ ] `--json` returns API-shaped changes object
- [ ] Handles 503 from storage gracefully with actionable error
- [ ] Unit tests for diff handler and JSON serialization
