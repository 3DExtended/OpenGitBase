## What to build

Add **`OpenGitBase.Cli.Integration.Tests`** coverage (extend existing project) for merge request CLI flows against the in-process API test host. At minimum: register user, create repo with branches/commits (or seed via API), **`ogb mr create`**, **`ogb mr list`**, **`ogb mr view`**, **`ogb mr status`**, and either **`ogb mr close`** or full **approve → merge** path if test data supports protected-branch defaults.

## Acceptance criteria

- [ ] Integration test runs via `dotnet test tests/OpenGitBase.Cli.Integration.Tests`
- [ ] Uses `CliApp.RunAsync` with overrides (in-memory credentials, dedicated HttpClient)
- [ ] Correct API base URL for in-process host (no `/api` prefix quirk)
- [ ] At least one lifecycle test: create → read → mutate (close or merge)
- [ ] Document any test-host limitations in item notes
