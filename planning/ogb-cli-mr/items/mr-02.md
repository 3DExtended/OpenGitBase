## What to build

Implement **`ogb mr view <number>`** to fetch and display a single merge request: title, body, status, draft flag, source/target refs, SHAs, approval counts, creator, timestamps, and web URL. Support **`--commits`** (or equivalent) to include the commit list from the commits endpoint. Human and `--json` output.

## Acceptance criteria

- [ ] `ogb mr view 7` calls GET merge request by number and prints metadata
- [ ] `--commits` includes commit series from GET `…/commits`
- [ ] `--json` emits full DTO (including identifier wrappers deserialized to GUIDs)
- [ ] 404 and 401/403 produce non-zero exit with clear message
- [ ] Unit tests for view handler and commits optional path
